# 启用 GitHub Pages 指南

## 📋 步骤说明

### 方法 1：通过 GitHub 网页界面启用（推荐）

1. **访问你的仓库**
   - 打开：https://github.com/xenon-w/FilterGS.github.io

2. **进入设置页面**
   - 点击仓库页面顶部的 **Settings**（设置）标签

3. **找到 Pages 设置**
   - 在左侧菜单中，向下滚动找到 **Pages** 选项
   - 点击进入 Pages 设置页面

4. **配置 Pages 源**
   - 在 "Source"（源）部分：
     - **Branch**（分支）：选择 `main`
     - **Folder**（文件夹）：选择 `/ (root)`（根目录）
   - 点击 **Save**（保存）按钮

5. **等待部署**
   - GitHub 会在几分钟内构建和部署你的网站
   - 页面顶部会显示一个绿色的提示框，显示部署状态
   - 部署完成后，会显示你的网站 URL

### 方法 2：通过 GitHub Actions（自动部署）

如果你的仓库已经配置了 Actions，GitHub 会自动部署。

## 🌐 访问你的网站

启用后，你的网站将在以下地址可用：

**主要 URL：**
- https://xenon-w.github.io/FilterGS.github.io/

**注意：**
- 如果仓库名是 `xenon-w.github.io`（用户名.github.io），则直接访问：
  - https://xenon-w.github.io/
- 你的仓库名是 `FilterGS.github.io`，所以需要使用完整路径

## ⏱️ 部署时间

- **首次部署**：通常需要 1-5 分钟
- **后续更新**：每次推送代码后，通常 1-3 分钟自动更新

## ✅ 验证部署

1. **检查部署状态**
   - 在仓库页面，点击 **Actions** 标签
   - 查看是否有 "pages build and deployment" 工作流
   - 绿色勾号表示部署成功

2. **访问网站**
   - 打开你的网站 URL
   - 确认页面正常显示

## 🔧 常见问题

### 问题 1：404 错误
- **原因**：部署尚未完成，或路径不正确
- **解决**：等待几分钟后重试，确认 URL 正确

### 问题 2：样式或资源加载失败
- **原因**：资源路径可能不正确
- **解决**：检查 `index.html` 中的资源路径是否使用相对路径（如 `./static/`）

### 问题 3：部署失败
- **原因**：可能有构建错误
- **解决**：
  - 检查 Actions 标签中的错误信息
  - 确认 `index.html` 在根目录
  - 检查文件结构是否正确

### 问题 4：更改后网站未更新
- **原因**：缓存或部署延迟
- **解决**：
  - 等待几分钟
  - 强制刷新浏览器（Ctrl+F5）
  - 检查 GitHub Actions 中的部署状态

## 📝 自定义域名（可选）

如果你想使用自定义域名：

1. 在 Pages 设置中添加你的域名
2. 在域名 DNS 中添加 CNAME 记录
3. 在仓库根目录创建 `CNAME` 文件

## 🎉 完成！

启用后，你的网站就可以通过互联网访问了！
